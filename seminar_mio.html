Diffusion models work in two stages. First, the forward process. Start with a clean image, this one is from the COCO dataset, and we add Gaussian noise step by step. Nothing learned here, it's pure math. We can noise any image to any timestep we want. What the model learns is how to go the other way.

Here's how. We call it "t-error." Take an image. Add noise at some timestep t. Ask the model to reconstruct the original in one step. Measure the squared error. [click] If the model was trained on this image, it reconstructs it well, low error. If it's never seen this image, the reconstruction is worse, higher error. That's the membership signal. Simple as that.

Gaussian QR: Precise Threshold Control

We have scores. How do we decide the threshold?

Fit a conditional Gaussian model in log-score space:

$\tau(\text{FPR}) = \mu(x) + \sigma(x) \cdot \Phi^{-1}(1 - \text{FPR})$

Key value: Closed-form threshold at any FPR level.
Control false positives at 0.1% or 0.01% — no empirical calibration needed.

So we have scores right. How do we pick a threshold? [click] We fit a Gaussian model in log-score space and get a closed-form threshold for any false positive rate. [click] The formula is on the slide, pretty straightforward. [click] What this gives us is precise control over false positives, like one in a thousand or one in ten thousand, just from a formula. No calibration set needed. Important when false accusations have real consequences.

From Detection to Verification

Not just “I recognize this image,”

but “statistical evidence that this model
was trained on my private data,
and public models were not.”
But we need more than per-image detection. We need to show that this model was trained on my data, and public models were not. That's the standard for anything legal.

Three-Point Verification Protocol

⟷

C1: Consistency

Suspect model ≈ owner model on evidence set.

Same lineage confirmed.

↕

C2: Separation

Owner model reconstructs evidence set much better than public reference models.
p < 10−6, Cohen's d > 2

×

C3: Ratio

Mean reference model error / mean owner error on evidence set.

Ratio > 5×

We use three criteria. All must hold. [click] C1, Consistency: the suspect model behaves like the owner model on the evidence set. Same lineage. [click] C2, Separation: the owner model reconstructs evidence images much better than any public reference model. We need p below ten to the minus six, Cohen's d above 2. Both statistical and practical significance. [click] C3, Ratio: the reference model's error divided by the owner's error on the evidence set has to be above 5x. A simple sanity check that the gap is real.

Verdict

All three criteria must hold simultaneously:

C1 ∧ C2 ∧ C3  →  VERIFIED
Any criterion fails  →  REJECTED

This is hypothesis testing, not a black-box classifier.
Every decision comes with p-values, effect sizes, and confidence intervals.

If all three pass, verified. If any one fails, rejected. [click] Every decision comes with p-values and effect sizes. It's hypothesis testing, not a black box. It's auditable and reproducible.

Discussion & Wrap-up

Scope & Future Work

White-box access — requires model weights. Standard in IP litigation & open-weight models.

Full retraining — if adversary trains from scratch, memorization is erased. But that costs as much as the original.

MIA = dual-use — privacy attack on individuals, IP defense for model owners. Same technique, opposite framing.

Open questions: black-box (API-only) verification · cross-architecture transfer (DiT, flow-matching) · adversarial memorization erasure · legal framework integration

[click] We do need white-box access, meaning the model weights. But that's standard in IP litigation, and open-weight models are open by definition. [click] If someone trains from scratch, we can't detect it. But that costs just as much as the original, so it defeats the purpose of stealing. [click] Here's an interesting duality. Membership inference is usually a privacy attack on individuals. We flip it into an IP defense tool for model owners. Same technique, opposite framing. [click] Open questions. Black-box verification. Cross-architecture transfer. Can someone deliberately erase the memorization signal. And how all of this maps to actual IP law.
